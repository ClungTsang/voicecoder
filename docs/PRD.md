> 版本：v1.0 · 日期：2026-06-09

## 1. 产品定位

**VoiceCoder Desktop** 是 VoiceCoder 语音输入工具的可视化管理界面。它不是另一个转录工具——它是 VoiceCoder 后台服务的控制面板，让用户可以配置模型、查看转写历史、管理设置，而无需编辑命令行或 JSON 文件。

### 核心原则
- **控制面板定位**：不替代命令行热键交互，而是提供可视化管理
- **轻量优先**：Electron 太重，用 Tauri（Rust 内核，~5MB 安装包）
- **跨平台**：macOS + Windows 共用一套 UI 代码
- **非 AI 风格**：暖白背景、teal 色系、内容驱动，与官网风格一致

---

## 2. 技术选型

| 层 | 方案 | 理由 |
|---|------|------|
| 框架 | **Tauri v2** | 安装包 ~5MB（Electron ~150MB），原生 WebView，跨平台 |
| 前端 | **Vue 3 + TypeScript** | 轻量、响应式、组件生态丰富 |
| UI 库 | **自定义 CSS**（参考官网风格） | 保持一致性，不引入重型 UI 框架 |
| 数据存储 | **SQLite**（via better-sqlite3 / tauri-plugin-sql） | 本地持久化，转写历史、配置、用户信息 |
| 认证 | **GitHub OAuth** | 用户已有 GitHub 账号（开源项目），无需注册 |
| 图标 | **Lucide Icons** | 轻量、线条风格、非 AI 味 |

### 3.2 仪表盘 (Dashboard)

**参考**：Buzz 首页统计 + MacWhisper 最近转写

- 服务状态指示灯（绿色 = 运行中 / 红色 = 已停止）
- 今日转写次数 / 总转写时长
- 最近 5 条转写记录（点击跳转详情）
- 模型加载状态（已加载/未加载/加载中）
- 快捷操作：「启动服务」「停止服务」「打开热键设置」

### 3.3 转写历史 (History)

**参考**：MacWhisper 历史列表 + Otter.ai 时间线

- 列表视图：日期 | 时间 | 文本预览 | 时长 | 操作（复制/删除/导出）
- 搜索框：全文搜索转写内容
- 筛选：按日期范围、按关键词
- 详情页：完整文本 + 音频波形（可选）+ 创建时间 + 字数统计
- 批量导出：TXT / JSON / CSV
- 每条记录可标记「收藏」

### 3.4 实时转写 (Live)

**参考**：Otter.ai 实时转写界面

- 实时麦克风音量指示器（波形图）
- 点击开始/停止录音按钮
- 实时显示转写文本（流式输出）
- 「复制全部」「复制最后一段」按钮
- 转写结果自动保存到历史

### 3.5 设置 (Settings)

#### 3.5.1 模型管理

**参考**：Buzz 模型选择器 + MacWhisper 模型下载

- **当前模型**：显示已加载的模型名称和大小
- **模型列表**：
  - SenseVoice Small（默认，~1.1 GB）— 推荐
  - SenseVoice Medium（~2.5 GB）— 更高准确率
  - Whisper Small（~461 MB）— 备选
  - Whisper Medium（~1.5 GB）— 备选
- **下载管理**：显示每个模型的下载进度、大小、已下载状态
- **一键切换**：选择模型 → 自动下载（如未下载）→ 重新加载服务
- **存储信息**：显示模型总占用空间

#### 3.5.2 语音引擎

- **输入设备**：下拉选择麦克风（默认：系统默认）
- **采样率**：显示当前采样率（44100Hz / 16000Hz）
- **音量归一化**：开关 + 放大倍数滑块（默认：自动检测）
- **VAD 灵敏度**：静音检测阈值（低/中/高）
- **语言**：简体中文（默认）/ English / 日本語

#### 3.5.3 热键设置

- **触发方式**：鼠标中键（默认）/ 自定义快捷键
- **热键延迟**：防抖时间（0.3s - 1.0s，默认 0.5s）

#### 3.5.4 技术名词纠错

- **纠错列表**：显示 tech_terms.json 的 87 个词条
- **添加词条**：输入错误 → 正确 → 添加
- **删除词条**：滑动删除
- **LLM 纠错**：开关（默认开启）+ API 配置
  - API 类型：New API Gateway / OpenAI / 自定义
  - API URL
  - API Key（密码输入框）
  - 测试连接按钮

#### 3.5.5 输出设置

- **自动粘贴**：开关（默认开启）
- **粘贴后操作**：无 / 自动加句号 / 自动加换行
- **剪贴板保留**：开关（转写后是否保留在剪贴板）

#### 3.5.6 服务管理

- **开机自启**：开关（默认开启）
- **崩溃恢复**：开关（默认开启）
- **日志级别**：INFO / DEBUG / WARNING
- **查看日志**：打开日志文件
- **重启服务**：按钮

### 3.6 账户 (Account)

- **GitHub 登录**：OAuth 按钮 → 浏览器跳转 → 回调授权
- **用户信息**：头像、用户名、GitHub 主页链接
- **同步设置**：（预留）是否同步配置到云端
- **登出**按钮

### 3.7 检查更新

- 当前版本号显示（底部侧边栏）
- 「检查更新」按钮 → 调用 GitHub Releases API
- 发现新版本：
  - 显示版本号 + 更新日志
  - 「下载更新」按钮（跳转 GitHub Release 页面）
  - macOS: 下载 .dmg
  - Windows: 下载 .msi / .exe
- 已是最新版本：✓ 提示

## 4. 数据库设计 (SQLite)

### 4.1 transcriptions 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增 ID |

| 字段 | 类型 | 说明 |

## 5. 页面流程

### 5.1 首次启动

```
启动 → 检测后端服务
  ├── 服务已运行 → 进入主界面
  └── 服务未运行 → 引导页
        ├── Step 1: 欢迎 + 介绍
        ├── Step 2: 下载模型（SenseVoice Small）
        ├── Step 3: 选择麦克风
        ├── Step 4: 测试录音（说一句话验证）
        └── Step 5: 完成 → 进入主界面
```

### 5.2 GitHub OAuth 流程

```
点击「GitHub 登录」
  → 打开浏览器 https://github.com/login/oauth/authorize?client_id=XXX
  → 用户授权
  → 回调到 localhost:port/callback?code=XXX
  → 后端用 code 换 access_token
  → 获取用户信息
  → 存入本地 SQLite
  → 更新 UI 显示头像和用户名
```

## 6. 非功能需求

| 项目 | 要求 |
|------|------|
| 安装包大小 | macOS < 10 MB, Windows < 10 MB |
| 启动时间 | < 2 秒 |
| 内存占用 | 空闲 < 80 MB（不含后端服务） |
| 兼容性 | macOS 14+ (Apple Silicon + Intel), Windows 10/11 (x64 + ARM64) |
| 无障碍 | 支持键盘导航、高对比度模式 |
| 国际化 | 首版仅简体中文，预留 i18n |

## 8. UI 设计规范

### 色彩（与官网一致）

| 用途 | 色值 | 说明 |
|------|------|------|
| 背景 | `#fafaf9` | 暖白 |
| 侧边栏 | `#f5f5f4` | 略深暖白 |
| 文字 | `#1c1917` | 石板黑 |
| 次要文字 | `#78716c` | 灰棕 |

- 正文：`-apple-system, 'Noto Sans SC', sans-serif`

### 布局

- 侧边栏宽度：240px（可折叠到 64px icon-only 模式）
- 内容区最大宽度：无限制（流式）
- 卡片圆角：8px
- 间距基准：8px（8 / 16 / 24 / 32 / 48）
